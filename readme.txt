
фиксированная верстка
не адаптивная

.container{
    width: 1170px;
    margin: 0 auto;
}
для ограничивающего контейнера устанавливается:
-конкретная ширина
-маржин 0 авто

при этом при уменьшении размера экрана
часть контента будет выходить за пределы экрана
т.е. будет не видна

если прописать мета тег:
<meta name='viewport' content='width=device-width,initial-scale=1'/>

и дополнительно для боди:
mix-width: 1170px;

то будет более-менее смотреться при разной ширине,
будет масштабироваться и уменьшаться вместе с экраном

такой спо



==================================================
ОТЗЫВЧИВАЯ ВЕРСТКА
резиновая



1. мкта тег
<meta name='viewport' content='width=device-width,initial-scale=1'/>

2. для боди убираем ширину совсем

3. для контейнера
вместо фиксированной ширины ставим максимальную ширину

max-width: 1170px;

ограничиваем по ширине, но не указываем явную ширину

4. и дальше по коду убираем конкретные фиксированные значения ширин для элементов
ширины указываются не фиксированные а процентные

хорошо использовать когда контентная часть не ограничивается контейнером













==================================================
адаптивная верстка
с брейкпоинтами

подстраивается под разные устройства и разные экраны
с помощью брейкпоинтов
медиазапросы

когда ширина экрана подходит к брейк поинту
ширина нашего контейнера становится у'же
и контент перестраивается

1. мкта тег
<meta name='viewport' content='width=device-width,initial-scale=1'/>

2. не должно быть никаких фиксированных значений ширин

3. пишем сетку в медиа запросах по брейкпоинтам для контейнера


@media (max-width: 1170px) {
 div.section-inner {
  max-width: 970px;
 }
}
@media (max-width: 992px) {
 div.section-inner {
  max-width: 750px;
 }
}
@media (max-width: 767px) {
 div.section-inner {
  max-width: none;
 }
}

отзывчиво-адаптивная верстка
резиновая с брейкпоинтами

совмещает в себе преимущества резиновой и адаптивной верстки



1. мкта тег
<meta name='viewport' content='width=device-width,initial-scale=1'/>

2. для боди убираем ширину совсем

3. для контейнера
вместо фиксированной ширины ставим максимальную ширину

max-width: 1170px;

ограничиваем по ширине, но не указываем явную ширину

4. и дальше по коду убираем конкретные фиксированные значения ширин для элементов
ширины указываются не фиксированные а процентные

5. в медиазапросах с брейкпоитнами
перестраиваем блоки контента
